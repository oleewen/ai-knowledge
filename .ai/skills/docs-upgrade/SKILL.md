# docs-upgrade（定向文档与注释升级）

本技能用于**可控范围内的文本一致化**：先完成主目标文件的增改，再按默认规则把关联处与同类表述对齐。复杂逻辑或运行时行为改动不在此技能范围内（除非用户明确要求同步注释/文档表述）。

## 何时必须启用

满足**任一**即按本技能执行（含直接编辑仓库文件）：

- 用户调用 `/docs-upgrade`，或上下文中附加 `@docs-upgrade` / `@.ai/skills/docs-upgrade`。
- 用户**明确列出**待修改或待核对的一个或多个文件路径。
- 用户**粘贴、引用或 @ 文件**并给出修改意图（对齐术语、改说明、替换片段、更新注释等）。

范围明显过大（例如无边界全库术语替换）时，先与用户确认批次或排除目录，再继续。

## 能力边界

| 包含 | 不包含（除非用户明确要求同步文档/注释） |
| ---- | ---------------------------------------- |
| 文档、纯文本、代码与配置中的**注释**、字符串内文档路径 | 纯业务逻辑重构、与文档无关的大规模代码改写 |
| 链式引用检索 + 关键词检索（同义/近义/中英文对应，见参考文） | 替代 `docs-change` 的变更索引、`docs-indexing` 的全库索引重建 |

## 替换简写（将左侧替换为右侧）

以下三种形式**等价**：在 trim 后把 `a` 替换为 `b`。分隔符与 `a`、`b` 之间**可以**有空格。

| 形式 | 含义 |
| ---- | ---- |
| `a - b` | `a` → `b` |
| `a > b` | `a` → `b` |
| `a 2 b` | `a` → `b`（数字 2 表示 to） |

规则：

- 若 `a` 或 `b` 含 `|`、换行、或边界不清，使用引号：`"旧术语" - "新术语"`。
- 替换前确认命中范围：若 `a` 在目标范围内不唯一，列出命中次数与示例行，或请用户确认是否全部替换。

## 主流程（顺序执行）

1. **锁定目标**：解析用户给出的路径；无路径时用搜索定位候选，**多候选则列出选项**后再编辑。
2. **主修改**：按用户意图完成增写、改写或替换（含注释与字符串中的文档性表述）。
3. **关联与语义同步**（见下节「默认与例外」）：在链式与关键词两条线上查找需对齐的同类表述并更新。
4. **回链校验**：若存在互链，确认相对路径、锚点与反引号路径仍有效。
5. **不确定项门禁**：无法核实的内容 → **停止并输出编号选项**（见「须由用户决策的情形」），待用户选择后再改。

## 链式与关键词：默认与例外

- **默认**：对本轮**已修改的每个文件**同时做：
  - **引用链检索**：出站（本文件引用了谁）与入站（谁引用了本文件），将关联处与本次变更**同一概念、重复表述、需对齐的句子或术语**一并修订。细则见 [references/related-doc-discovery.md](references/related-doc-discovery.md)。
  - **关键词 / 语义检索**：当文件间无直接链接但存在同义/近义/中英文对应表述、复制粘贴段、或同一术语散落多处时，用 `rg` 等与仓库约定的方式**分轮检索各候选词面**，按 [references/semantic-keyword-discovery.md](references/semantic-keyword-discovery.md) 判断是否应改。
- **例外**：用户明确说「只改本文件」「不要同步关联文档/注释」「不要全库搜」时，**跳过**扩展检索，仅改指定范围。

若用户未声明例外且存在歧义，可简短确认：「本次是否需要同步引用链上的其他文件，并对全库同类表述做关键词检索？」

## 须由用户决策的情形

下列情况**不得编造**；应输出**编号选项**（可附简短推荐与理由），待用户选择后再写入：

- 业务规则、指标数字、日期、责任人、合规/政策结论、架构决策陈述。
- 是否删除过时段落、是否保留兼容说明、标题层级或文档结构取舍。
- 多文件同名、多段相似但语义可能不同的文本、或链式/关键词检索下「是否视为同一概念」存疑时。

## 验证

- 抽查：链接可解析、表格与代码块未破坏、术语与主目标一致。
- 用户要求时运行仓库既有校验命令。
