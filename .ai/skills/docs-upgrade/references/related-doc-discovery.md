# 关联文档发现（供 docs-upgrade 按需加载）

仅在 **修改模式**下使用本节；**新增模式**只改指定文件，不执行入站/出站检索。

## 出站（主文档引用了谁）

1. 读主文件全文，收集：
   - Markdown 链接：`[text](path)`、`[text](path#anchor)`
   - 裸路径：反引号内的 `*.md`、`docs/...`
2. 将相对路径解析为仓库内绝对路径（相对当前文件目录）。

## 入站（谁引用了主文档）

对主文档的每一种可检索键做检索（至少包含）：

| 键类型 | 示例 | 工具 |
|--------|------|------|
| 仓库内相对路径 | `docs/solutions/SOLUTION-260403-foo.md` | `rg -F` |
| 从仓库根起算路径 | `solutions/SOLUTION-260403-foo.md` | `rg -F` |
| 文件名 | `SOLUTION-260403-foo.md` | `rg -F`（注意同名文件需人工区分） |
| INDEX/README 中的显示名 | 若用户以标题指代 | `rg` 标题关键词 |

注意：

- 排除 `target/`、二进制目录及生成物（与仓库 AGENTS 约定一致）。
- 命中过多时：按路径前缀 `docs/` 优先，再请用户缩小范围。

## 同类表述

「同类表述」指：与主文档同一术语定义、同一列表项、同一摘要句、或明显从主文档复制的段落。不要强行统一仅碰巧用词相近但含义不同的段落。

## 链式深度

默认：**主文档 → 一层入站 + 一层出站**；若用户要求「全链」或「所有引用」，再扩展 BFS，并提示工作量与风险。
