# 质量标准与扫描规则

## 扫描规则

### 路径解析

- 基于项目根目录的相对路径
- 输出路径优先级：用户指定 > `./system/` > `./doc/` > `./INDEX_GUIDE.md`

### 文件过滤

自动忽略以下路径：

| 排除模式 | 原因 |
|----------|------|
| `target/`、`build/` | 构建产物 |
| `.git/` | 版本控制 |
| `node_modules/` | 前端依赖 |
| `*.log` | 运行时日志 |
| `*.class`、`*.jar` | 二进制文件 |
| `.idea/`、`.vscode/` | IDE 配置 |

### 深度控制

| 深度 | 扫描对象 | 典型文件类型 |
|------|----------|--------------|
| 1（拓扑） | 目录结构、`pom.xml`/`build.gradle`、`README`、`*.yml`/`*.yaml` | 配置/元数据 |
| 2（结构） | +`*.java`/`*.kt` 类名/函数名/注释、接口定义、`*.xml` Mapper | 源码结构 |
| 3（精读） | +源码全文（含判断分支、内联注释）、业务逻辑、SQL | 源码全文 |

### MECE 原则

- 每个文件只归入一个章节
- 章节之间互斥穷尽
- 交叉引用以超链接形式体现，不重复内容

## 质量验证清单

### 结构完整性

- [ ] 九章均有实质内容（无空章节）
- [ ] 章节编号连续，无遗漏
- [ ] 小节层级统一（最多三级标题）

### 信息密度

- [ ] 每条目 15-30 字，无空话套话
- [ ] 表格行有实际数据，非占位符
- [ ] 代码片段精简，仅保留关键签名

### 准确度

- [ ] 只索引实际读取的内容（零幻觉原则）
- [ ] 路径可访问、类名/方法名与源码一致
- [ ] 版本号、配置值来自实际文件

### 可追溯性

- [ ] 操作日志已追加到 `indexing-log.jsonl`
- [ ] 日志包含模式、深度、文件数、耗时
- [ ] 增量模式记录基线时间戳

### 交叉引用

- [ ] 模块间依赖引用正确
- [ ] 文档内链接可点击且指向正确位置
- [ ] 与 `KNOWLEDGE_INDEX.md`、`README.md`、`AGENTS.md` 的交叉引用一致

---

## 增量模式规则

1. 从 `indexing-log.jsonl` 读取上次 `indexing_finished_at` 作为基线
2. 仅扫描基线时间之后变更的文件
3. 对已有章节：合并更新，不清空未变更部分
4. 对新增文件：归入对应章节
5. 对删除文件：从索引中移除相关条目
6. 最终结构须与全量索引保持一致

## 错误处理

| 场景 | 处理 |
|------|------|
| 无历史日志 | 强制 full 模式，提示用户 |
| 变更索引缺失 | 强制 full 模式 |
| 输出目录不存在 | 自动创建目录 |
| 文件不可读 | 跳过并记录到索引边界（§八） |
| 增量基线无效 | 降级为 full 模式 |
