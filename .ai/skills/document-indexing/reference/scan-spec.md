# 扫描执行规范

document-indexing 技能的执行细节：深度级别定义、模式逻辑、文件过滤、增量规则、日志格式与错误处理。

---

## 深度级别

| 级别 | 名称 | 扫描范围 | 适用场景 |
|------|------|----------|----------|
| 1 | 拓扑扫描 | 目录结构、主要模块、元数据 | 快速概览、大型仓库初扫 |
| 2 | 结构分析 | +类名/函数名/注释、接口定义、关键配置 | 日常索引、增量更新 |
| 3 | 精读提取 | +源码全文、判断分支、业务逻辑 | 逆向工程、全量索引 |

### 各深度扫描对象

| 深度 | 扫描对象 | 典型文件类型 |
|------|----------|--------------|
| 1（拓扑） | 目录结构、`pom.xml`/`build.gradle`、`README`、`*.yml`/`*.yaml` | 配置/元数据 |
| 2（结构） | +`*.java`/`*.kt` 类名/函数名/注释、接口定义、`*.xml` Mapper | 源码结构 |
| 3（精读） | +源码全文（含判断分支、内联注释）、业务逻辑、SQL | 源码全文 |

---

## 模式逻辑

### full（全量）

忽略历史记录，全量扫描代码库。适用于首次索引或需要重建索引时。

### incremental（增量）

基于 `indexing-log.jsonl` 上次完成时间戳，仅处理变更文件：

1. 从 `indexing-log.jsonl` 读取最后一条记录的 `indexing_finished_at` 作为基线
2. 仅扫描基线时间之后变更的文件
3. 对已有章节：合并更新，不清空未变更部分
4. 对新增文件：归入对应章节
5. 对删除文件：从索引中移除相关条目
6. 最终结构须与全量索引保持一致

无历史记录时自动降级为 full。

## 路径解析

- 基于项目根目录的相对路径
- 输出路径优先级：用户指定 > `./system/` > `./doc/` > `./INDEX_GUIDE.md`

---

## 文件过滤

自动忽略以下路径：

| 排除模式 | 原因 |
|----------|------|
| `target/`、`build/` | 构建产物 |
| `.git/` | 版本控制 |
| `node_modules/` | 前端依赖 |
| `*.log` | 运行时日志 |
| `*.class`、`*.jar` | 二进制文件 |
| `.idea/`、`.vscode/` | IDE 配置 |

---

## 日志格式

每次执行完毕后追加（append）一条到 `changelogs/indexing-log.jsonl`：

```json
{
  "timestamp": "ISO-8601",
  "data_mode": "full|incremental",
  "read_mode": 1,
  "output_path": "./system/INDEX_GUIDE.md",
  "indexed_files": 156,
  "duration_ms": 45230,
  "indexing_finished_at": 1704115200000
}
```

必需字段：`timestamp`、`data_mode`、`read_mode`、`output_path`、`indexed_files`、`duration_ms`、`indexing_finished_at`。

## 错误处理

| 场景 | 处理 |
|------|------|
| 无历史日志 | 强制 full 模式，提示用户 |
| 变更索引缺失 | 强制 full 模式 |
| 输出目录不存在 | 自动创建目录 |
| 文件不可读 | 跳过并记录到索引边界（§八） |
| 增量基线无效 | 降级为 full 模式 |
