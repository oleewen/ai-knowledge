# 质量验证标准

document-indexing 技能的产出质量验证清单。步骤 5（质量验证）时逐项核对。

---

## 结构完整性

- [ ] 九章均有实质内容（无空章节）
- [ ] 确实无内容的小节标注 `[未索引]` 并说明原因，归入 §八排除列表
- [ ] 章节编号连续，无遗漏
- [ ] 小节层级统一（最多三级标题）

## 信息密度

- [ ] 每条目 15-30 字，无空话套话
- [ ] 表格行有实际数据，非占位符
- [ ] 代码片段精简，仅保留关键签名

## MECE 原则

- [ ] 每个文件只归入一个最匹配的章节
- [ ] 章节之间互斥穷尽
- [ ] 交叉引用以超链接形式体现，不重复内容

## 准确度

- [ ] 只索引实际读取的内容（零幻觉原则）
- [ ] 路径可访问、类名/方法名与源码一致
- [ ] 版本号、配置值来自实际文件
- [ ] 所有路径为项目根相对路径

## 可追溯性

- [ ] 操作日志已追加到 `indexing-log.jsonl`
- [ ] 日志包含模式、深度、文件数、耗时、`indexing_finished_at`
- [ ] 增量模式记录基线时间戳

## 交叉引用

- [ ] 模块间依赖引用正确
- [ ] 文档内链接可点击且指向正确位置
- [ ] 与 `KNOWLEDGE_INDEX.md`、`README.md`、`AGENTS.md` 的交叉引用一致

## 深度 3（精读）专项

在用户确认的深度为 **3** 时，除上述通用项外须额外核对：

- [ ] 已按 [scan-spec.md](scan-spec.md)「深度 3 应读尽读准则」遍历可读目录，未无故跳过同级子目录或同类文件
- [ ] 索引条目数量与路径覆盖面**明显高于**同库在深度 2 下的合理预期（在九章与 MECE 前提下尽可能多）
- [ ] 凡未读、部分读或排除的路径均在 §八（或边界说明）中有记录，正文中无将未读内容当作已核实事实的表述
