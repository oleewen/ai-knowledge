# 质量验证标准

document-indexing 技能的产出质量验证清单。步骤 5（质量验证）时逐项核对。

---

## 结构完整性

- [ ] 九章均有实质内容（无空章节）
- [ ] 确实无内容的小节标注 `[未索引]` 并说明原因，归入 §八排除列表
- [ ] 章节编号连续，无遗漏
- [ ] 小节层级统一（最多三级标题）

## 信息密度

- [ ] 每条目 15-30 字，无空话套话
- [ ] 表格行有实际数据，非占位符
- [ ] 代码片段精简，仅保留关键签名

## MECE 原则

- [ ] 每个文件只归入一个最匹配的章节
- [ ] 章节之间互斥穷尽
- [ ] 交叉引用以超链接形式体现，不重复内容

## 准确度

- [ ] 只索引实际读取的内容（零幻觉原则）
- [ ] 路径可访问、类名/方法名与源码一致
- [ ] 版本号、配置值来自实际文件
- [ ] 所有路径为项目根相对路径

## 可追溯性

- [ ] 操作日志已追加到 `indexing-log.jsonl`
- [ ] 日志包含模式、深度、文件数、耗时、`indexing_finished_at`
- [ ] 增量模式记录基线时间戳

## 交叉引用

- [ ] 模块间依赖引用正确
- [ ] 文档内链接可点击且指向正确位置
- [ ] 与 `KNOWLEDGE_INDEX.md`、`README.md`、`AGENTS.md` 的交叉引用一致
