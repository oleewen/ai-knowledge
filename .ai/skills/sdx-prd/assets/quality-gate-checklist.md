# PRD 质量门禁清单

步骤 5 验收模板。Agent 在执行文档输出与评审时复制本清单并逐项勾选。

## 完整性

- [ ] 产品概述（§1）：产品目标引用需求分析文档与 MVP 范围，涉及角色完整
- [ ] 核心业务流程（§2.1）：主流程 Mermaid 图完整，步骤说明表齐全
- [ ] 分支异常流程（§2.2）：异常场景 EX-n 清单完整，处理方式与用户提示明确
- [ ] 系统交互流程（§2.3）：跨系统调用时序图完整（或标注「不适用」）
- [ ] 产品交互（§3）：交互流程与交互设计说明完整
- [ ] 用户故事清单（§4.1）：MVP 范围内每个 FR-n 至少有一个 US-n
- [ ] 用户故事详述（§4.2）：每个 US-n 有验收标准（Given-When-Then），至少含正常+异常场景
- [ ] 用例图（§5.1）：覆盖所有角色与核心交互
- [ ] 用例详述（§5.2）：每个 UC-n 有参与者、前后置条件、主成功场景、扩展场景
- [ ] 功能模块（§6）：模块划分完整，模块间关系图清晰
- [ ] 业务规则汇总（§7）：所有 BR-n 集中汇总，字段完整
- [ ] 数据字典（§8）：业务术语与状态定义完整
- [ ] 验收标准汇总（§9）：功能验收 AC-n 与非功能验收 NAC-n 齐全

## 一致性

- [ ] US-n 与需求分析 FR-n 一一对应，无遗漏
- [ ] BR-n 与需求分析 BR-n 及 `knowledge/business/` 无矛盾
- [ ] 业务流程与需求分析中的核心场景描述一致
- [ ] US-n / UC-n / BR-n / EX-n / AC-n / NAC-n 编号连续无断号
- [ ] 各章节交叉引用自洽（故事→用例→规则→验收）
- [ ] 用例与用户故事双向映射完整

## 可行性

- [ ] 用户故事粒度合理，可在单迭代内交付
- [ ] 业务流程无逻辑死循环或不可达状态
- [ ] 验收标准可度量、可测试
- [ ] 交互设计在现有技术框架下可实现

## 可追溯性

- [ ] 每个 US-n 可追溯到需求分析 FR-n
- [ ] 每个 UC-n 可追溯到 US-n
- [ ] 每个 BR-n 可追溯到需求分析 BR-n
- [ ] 每个 AC-n / NAC-n 可追溯到 US-n 或非功能需求
- [ ] 每个 EX-n 可追溯到业务流程步骤

## 格式规范

- [ ] frontmatter 完整（id、title、version、status、created、updated、parent、mvp_phase）
- [ ] 章节结构与 prd-template.md 一致，编号连续
- [ ] Mermaid 图表语法正确，可正常渲染
- [ ] 表格格式规范，列宽合理
- [ ] 术语表（§8.1）涵盖本文档专用术语
- [ ] 变更历史（§10.2）记录初始版本
