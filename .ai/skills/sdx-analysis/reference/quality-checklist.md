## 受众与语言

- [ ] 正文符合 [../../sdx-solution/reference/audience-and-language.md](../../sdx-solution/reference/audience-and-language.md)：产品/需求分析师与业务方可独立理解主流程

## 一致性

- [ ] 功能需求 FR-n 与解决方案 G-n 可追溯对应，无遗漏
- [ ] 业务规则 BR-n 与 `knowledge/business/` 无矛盾（或已标注分析盲区）
- [ ] MVP 拆分与优先级一致（P0 需求落在合理 MVP）
- [ ] 文档内部 FR-n / BR-n / R-n / MVP-n 编号连续无断号
- [ ] 各章节交叉引用自洽（需求→规则→MVP→风险）

## 可行性

- [ ] MVP 拆分粒度合理，每个可独立交付与演示
- [ ] 依赖关系无循环，MVP 间单向依赖
- [ ] 工作量估算有参照基准
- [ ] 高风险项（R-n）有具体应对策略与可跟进行动

## 可追溯性

- [ ] 每个 FR-n 可追溯到解决方案 G-n
- [ ] 每个 BR-n 可追溯到 FR-n
- [ ] 每个 MVP 可追溯到 FR 列表
- [ ] 每个 R-n 可追溯到依赖分析或影响面评估

## 格式规范

- [ ] 文末「## 文档元数据」内 YAML 完整（id、title、version、status、created、updated、author、reviewers、parent、tags）；文件开头无 `---` YAML frontmatter
- [ ] 章节结构与 [../assets/analysis-template.md](../assets/analysis-template.md) **六章**一致，编号连续
- [ ] 表格格式规范，列宽合理
- [ ] 术语表（§6.1）涵盖本文档专用术语
- [ ] 参考文档（§6.2）列出引用的解决方案/知识库文件
