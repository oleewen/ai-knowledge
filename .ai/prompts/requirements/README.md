# 需求交付阶段规范

需求交付阶段是将需求分析中各个 MVP 阶段的需求，以需求研发的形式，采用 SDD 规范逐个交付的核心执行阶段。每个 MVP 依次经历：产品需求 → 方案设计 → 需求开发三个子阶段。

本目录根目录存放阶段规范与模板；具体需求交付物按 `REQUIREMENT-{ID}/` 子目录组织，详见 `docs/README.md` 文档总体结构。
