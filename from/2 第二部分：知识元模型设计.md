## 第二部分：知识元模型设计

### 2.1 四视角元模型总览

本节定义每个视角的核心知识实体及其属性。所有实体均遵循统一的元数据规范（详见第六部分）。

#### 2.1.1 业务视角元模型 (Business View Meta-Model)

```yaml
# 业务视角核心实体关系
BusinessDomain:                    # 业务域
  contains: [BusinessSubDomain]
  attributes:
    - id: string                   # 唯一标识，如 "BD-ORDER"
    - name: string                 # 业务域名称
    - description: text            # 业务域描述
    - owner: string                # 业务负责人
    - strategic_classification: enum  # 核心域/支撑域/通用域

BusinessSubDomain:                 # 业务子域
  belongs_to: BusinessDomain
  contains: [BoundedContext]
  attributes:
    - id: string                   # 如 "BSD-ORDER-FULFILLMENT"
    - name: string
    - description: text
    - domain_expert: string        # 领域专家

BoundedContext:                    # 限界上下文
  belongs_to: BusinessSubDomain
  contains: [Aggregate]
  collaborates_with: [BoundedContext]  # 上下文映射
  attributes:
    - id: string                   # 如 "BC-ORDER-MGMT"
    - name: string
    - description: text
    - ubiquitous_language: map     # 统一语言词汇表
    - context_map_relations:       # 上下文映射关系
        - target_context: string
        - relation_type: enum      # ACL/OHS/CF/SK/Partnership 等
        - integration_pattern: string

Aggregate:                         # 聚合
  belongs_to: BoundedContext
  has_root: AggregateRoot
  contains: [DomainEntity, ValueObject, DomainEvent]
  attributes:
    - id: string                   # 如 "AGG-ORDER"
    - name: string
    - invariants: [string]         # 业务不变量/规则
    - lifecycle_description: text

AggregateRoot:                     # 聚合根
  is_a: DomainEntity
  attributes:
    - id: string
    - name: string
    - identity_field: string       # 身份标识字段
    - key_behaviors: [string]      # 核心行为列表

DomainEntity:                      # 领域实体
  attributes:
    - id: string
    - name: string
    - properties: [Property]
    - behaviors: [string]

ValueObject:                       # 值对象
  attributes:
    - id: string
    - name: string
    - properties: [Property]
    - equality_definition: string  # 相等性定义

DomainEvent:                       # 领域事件
  produced_by: Aggregate
  consumed_by: [BoundedContext]
  attributes:
    - id: string
    - name: string
    - payload_schema: object
    - business_meaning: text
    - trigger_condition: text
```

#### 2.1.2 产品视角元模型 (Product View Meta-Model)

```yaml
ProductLine:                       # 产品线
  contains: [ProductModule]
  attributes:
    - id: string                   # 如 "PL-ECOMMERCE"
    - name: string
    - description: text
    - target_users: [string]       # 目标用户群
    - product_owner: string

ProductModule:                     # 产品模块
  belongs_to: ProductLine
  contains: [Feature]
  attributes:
    - id: string                   # 如 "PM-SHOPPING-CART"
    - name: string
    - description: text
    - module_type: enum            # 前台/中台/后台

Feature:                           # 功能点
  belongs_to: ProductModule
  realized_by: [UseCase]
  attributes:
    - id: string                   # 如 "FT-ADD-TO-CART"
    - name: string
    - description: text
    - priority: enum               # P0/P1/P2/P3
    - status: enum                 # 规划中/开发中/已上线/已下线
    - acceptance_criteria: [string]

UseCase:                           # 用例
  belongs_to: Feature
  triggers: [BusinessProcess]
  attributes:
    - id: string                   # 如 "UC-PLACE-ORDER"
    - name: string
    - actor: string                # 参与者
    - preconditions: [string]
    - postconditions: [string]
    - main_flow: [Step]            # 主流程
    - alternative_flows: [Flow]    # 备选流程
    - exception_flows: [Flow]      # 异常流程

BusinessProcess:                   # 业务流程
  triggered_by: [UseCase]
  involves: [BoundedContext]       # 跨视角引用
  attributes:
    - id: string                   # 如 "BP-ORDER-TO-DELIVERY"
    - name: string
    - description: text
    - process_type: enum           # 核心流程/支撑流程/管理流程
    - steps: [ProcessStep]
    - sla: object                  # 服务水平协议
    - bpmn_ref: string             # BPMN 文件引用

BusinessRule:                      # 业务规则
  applies_to: [Feature, UseCase, BusinessProcess]
  attributes:
    - id: string                   # 如 "BR-ORDER-AMOUNT-LIMIT"
    - name: string
    - description: text
    - rule_type: enum              # 约束规则/推导规则/触发规则
    - formal_expression: string    # 形式化表达（可选）
    - source: string               # 规则来源（法规/业务决策等）
    - effective_date: date
    - expiry_date: date            # 可选
```

#### 2.1.3 技术视角元模型 (Technical View Meta-Model)

```yaml
System:                            # 系统
  contains: [Subsystem]
  attributes:
    - id: string                   # 如 "SYS-ECOMMERCE-PLATFORM"
    - name: string
    - description: text
    - system_type: enum            # 业务系统/平台系统/基础设施
    - tech_stack_summary: object

Subsystem:                         # 子系统
  belongs_to: System
  contains: [Application]
  attributes:
    - id: string                   # 如 "SUBSYS-ORDER"
    - name: string
    - description: text
    - responsibility: text         # 职责描述

Application:                       # 应用（对应代码仓库）
  belongs_to: Subsystem
  hosts: [Microservice]
  attributes:
    - id: string                   # 如 "APP-ORDER-SERVICE"
    - name: string
    - description: text
    - repo_url: string             # 代码仓库地址
    - tech_stack: object           # 技术栈详情
    - deployment_unit: string      # 部署单元标识
    - runtime_env: [string]        # 运行环境
    - ci_cd_pipeline: string       # CI/CD 流水线标识
    - docs_manifest_path: string   # 应用级文档清单路径

Microservice:                      # 微服务
  deployed_on: Application
  belongs_to_context: BoundedContext  # 跨视角引用
  exposes: [APIEndpoint]
  consumes: [APIEndpoint]
  publishes: [DomainEvent]
  subscribes: [DomainEvent]
  attributes:
    - id: string                   # 如 "MS-ORDER-COMMAND"
    - name: string
    - description: text
    - service_type: enum           # 命令服务/查询服务/聚合服务/网关
    - communication_style: enum    # 同步/异步/混合
    - resilience_patterns: [string] # 熔断/限流/重试等

APIEndpoint:                       # 接口端点
  belongs_to: Microservice
  attributes:
    - id: string                   # 如 "API-CREATE-ORDER"
    - path: string                 # 接口路径
    - method: string               # HTTP 方法 / RPC 方法
    - protocol: enum               # REST/gRPC/GraphQL/MQ
    - request_schema: object
    - response_schema: object
    - openapi_ref: string          # OpenAPI 规约引用
    - asyncapi_ref: string         # AsyncAPI 规约引用（异步接口）
    - sla: object                  # QPS/延迟/可用性
    - version: string
    - deprecation_info: object     # 废弃信息

InfraComponent:                    # 基础设施组件
  used_by: [Application]
  attributes:
    - id: string
    - name: string
    - type: enum                   # 数据库/缓存/消息队列/网关/注册中心等
    - product: string              # 具体产品如 MySQL/Redis/Kafka
    - version: string
    - configuration_ref: string    # 配置文件引用
```

#### 2.1.4 数据视角元模型 (Data View Meta-Model)

```yaml
DataEntity:                        # 数据实体
  maps_to: [DomainEntity, AggregateRoot]  # 跨视角引用
  stored_in: DataStore
  attributes:
    - id: string                   # 如 "DE-T_ORDER"
    - logical_name: string         # 逻辑名称
    - physical_name: string        # 物理表名/集合名
    - description: text
    - fields: [DataField]
    - indexes: [Index]
    - partitioning_strategy: string
    - data_volume_estimate: string # 数据量级估算
    - retention_policy: string     # 数据保留策略
    - sensitivity_level: enum      # 公开/内部/机密/绝密

DataField:                         # 数据字段
  attributes:
    - name: string
    - logical_name: string
    - data_type: string
    - nullable: boolean
    - default_value: string
    - constraints: [string]
    - sensitivity: enum
    - business_meaning: text       # 业务含义

DataStore:                         # 数据存储
  hosted_on: InfraComponent
  attributes:
    - id: string                   # 如 "DS-ORDER-DB"
    - name: string
    - store_type: enum             # RDBMS/NoSQL/搜索引擎/文件存储/数据仓库
    - product: string
    - schema_ref: string           # Schema 文件引用
    - owner_application: string    # 所属应用

DataFlow:                          # 数据流
  from: DataStore | Microservice
  to: DataStore | Microservice
  attributes:
    - id: string                   # 如 "DF-ORDER-TO-WAREHOUSE"
    - name: string
    - description: text
    - flow_type: enum              # 实时/准实时/批量/CDC
    - trigger: string              # 触发条件
    - transformation_rules: [string] # 数据转换规则
    - frequency: string            # 频率
    - data_entities_involved: [string] # 涉及的数据实体
    - pipeline_ref: string         # 数据管道配置引用

DataDictionary:                    # 数据字典条目
  attributes:
    - id: string
    - term: string                 # 术语
    - definition: text             # 定义
    - synonyms: [string]           # 同义词
    - data_type: string
    - value_range: string          # 取值范围
    - business_owner: string
    - source_of_truth: string      # 权威数据源
```

### 2.2 统一实体基础属性

所有知识实体均继承以下基础属性：

```yaml
KnowledgeEntityBase:
  id: string                       # 全局唯一标识
  name: string                     # 人类可读名称
  version: string                  # 文档版本
  status: enum                     # draft/review/approved/deprecated
  created_at: datetime
  updated_at: datetime
  created_by: string
  updated_by: string
  tags: [string]                   # 自由标签
  classification_tags:             # 分类标签（详见第六部分）
    domain: string
    subdomain: string
    bounded_context: string
    subsystem: string
    application: string
  cross_references: [CrossRef]     # 交叉引用列表
  completeness_score: float        # 完整度评分 0-1
  confidence_level: enum           # high/medium/low/unverified
```
