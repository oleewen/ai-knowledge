# ADR-001: 知识库结构与治理规范

## 状态

已通过 (Accepted)

## 上下文 (Context)

我们的系统业务复杂且处于不断演进中，原有的文档管理方式存在严重的信息碎片化、文档与代码脱节的问题。亟需一套能够整合业务、产品、技术、数据四个视角的知识库。

## 决策 (Decision)

1. 采用 **单一事实源 (SSOT)**，所有核心知识点仅在一处定义，通过全局唯一的 ID 引用。
2. 建立 **knowledge/** 下 **business**, **product**, **technical**, **data** 四视角目录，分别存放不同视角的模型和设计。
3. 弃用复杂的独立映射矩阵文件，改为在各层级的 `_meta.yaml` 文件中直接引用目标 ID 的方式来建立映射关联。
4. 系统级仓库负责宏观架构与跨域引用，应用级仓库负责微观实现，并在 CI/CD 阶段上报 `manifest.yaml` 更新系统级索引。

## 后果 (Consequences)

**正面**:

- 减少了冗余映射文件的维护成本。
- 明确了知识产出的归属权和生命周期管理。
- 为未来的自动化校验与依赖分析打下基础。

**负面**:

- 需要一定的学习成本，团队需要适应以 YAML 和 Markdown 结合的文档组织方式。
- 对各应用的 CI/CD 流水线提出了必须生成和上报 `manifest.yaml` 的强制要求。
