# 扫描执行规范

docs-indexing 技能的执行细节：深度级别定义、模式逻辑、文件过滤、增量规则、日志格式与错误处理。

---

## 深度级别

| 级别 | 名称 | 扫描范围 | 适用场景 |
|------|------|----------|----------|
| 1 | 拓扑扫描 | 目录结构、主要模块、元数据 | 快速概览、大型仓库初扫 |
| 2 | 结构分析 | +类名/函数名/注释、接口定义、关键配置 | 日常索引、增量更新 |
| 3 | 精读提取 | 在排除规则内**目录与文件应读尽读**，**尽量读全**可读文本；据此建立**尽可能多的**索引条目与交叉引用 | 权威全量地图、深度 Onboarding、RAG 证据最大化 |

### 各深度扫描对象

| 深度 | 扫描对象 | 典型文件类型 |
|------|----------|--------------|
| 1（拓扑） | 目录结构、`pom.xml`/`build.gradle`、`README`、`*.yml`/`*.yaml` | 配置/元数据 |
| 2（结构） | +`*.java`/`*.kt` 类名/函数名/注释、接口定义、`*.xml` Mapper | 源码结构 |
| 3（精读） | 在 1/2 基础上，对纳入扫描集的文件**完整读取**（或按文件类型规定的最大可读范围），覆盖业务逻辑、分支、注释、SQL、配置与文档正文 | 源码与文档全文为主 |

### 深度 3（精读）应读尽读准则

在用户已确认 `depth=3` 时，Agent 与脚本执行须遵守：

1. **目录遍历**：在「文件过滤」排除项之外，按仓库目录树**系统遍历**，不因体量或耗时跳过整棵子树；若某路径因权限、二进制、体积等**刻意不读**，须在产出 INDEX §八（或等价「边界/排除」小节）写明原因与路径。
2. **文件读取**：对扫描规范覆盖的扩展名与文本类型，**应读尽读**——优先读取**整文件**；仅在单文件过大或工具单次读取有硬限制时，允许分段读取，但须在索引中说明「分段/部分读取」及覆盖范围，且仍应覆盖该文件内**尽可能多的**可索引符号、配置键与逻辑块。
3. **索引最大化**：在九章结构与 MECE 分类下，**尽可能多**地生成独立条目（路径、职责摘要、关键 API、依赖与交叉链接）；禁止以「代表性抽样」为由只索引每目录下的少量文件而同目录其余文件未读。
4. **与深度 1/2 的关系**：深度 3 不是「比 2 多读一点」，而是**在允许范围内读全、写满**；仍遵守零幻觉——未实际读取的内容不得写成已核实事实。

## 模式逻辑

### full（全量）

忽略历史记录，全量扫描代码库。适用于首次索引或需要重建索引时。

### incremental（增量）

基于 `indexing-log.jsonl` 上次完成时间戳，仅处理变更文件：

1. 从 `indexing-log.jsonl` 读取最后一条记录的 `indexing_finished_at` 作为基线
2. 仅扫描基线时间之后变更的文件
3. 对已有章节：合并更新，不清空未变更部分
4. 对新增文件：归入对应章节
5. 对删除文件：从索引中移除相关条目
6. 最终结构须与全量索引保持一致

无历史记录时，须向用户说明增量前提不满足，请用户确认改走全量或中止，不得自动降级。

## 路径解析

- 基于项目根目录的相对路径
- 输出路径优先级：用户指定 > `./application/` > `./doc/` > `./INDEX_GUIDE.md`

---

## 文件过滤

自动忽略以下路径：

| 排除模式 | 原因 |
|----------|------|
| `target/`、`build/` | 构建产物 |
| `.git/` | 版本控制 |
| `node_modules/` | 前端依赖 |
| `*.log` | 运行时日志 |
| `*.class`、`*.jar` | 二进制文件 |
| `.idea/`、`.vscode/` | IDE 配置 |

---

## 日志格式

每次执行完毕后追加（append）一条到 `changelogs/indexing-log.jsonl`：

```json
{
  "timestamp": "ISO-8601",
  "data_mode": "full|incremental",
  "read_mode": 1,
  "output_path": "./application/INDEX_GUIDE.md",
  "indexed_files": 156,
  "duration_ms": 45230,
  "indexing_finished_at": 1704115200000
}
```

必需字段：`timestamp`、`data_mode`、`read_mode`、`output_path`、`indexed_files`、`duration_ms`、`indexing_finished_at`。

## 错误处理

| 场景 | 处理 |
|------|------|
| 无历史日志 | 说明增量前提不满足，请用户确认改走全量或中止 |
| 变更索引缺失 | 说明增量前提不满足，请用户确认改走全量或中止 |
| 输出目录不存在 | 自动创建目录 |
| 文件不可读 | 跳过并记录到索引边界（§八） |
| 增量基线无效 | 说明基线无效，请用户确认改走全量或中止 |
