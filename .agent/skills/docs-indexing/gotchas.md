# docs-indexing 常见陷阱

所有模式（full/incremental）、深度（1/2/3）及影响行为的参数（`--output`、`--since`）须满足 SKILL.md「用户确认门禁」。以下是对该门禁的补充场景说明。

---

## 参数门禁

**无历史日志仍强行增量，或 Agent 擅自改为全量**：`indexing-log.jsonl` 不存在或为空时，说明增量前提不满足，请用户确认改走全量或中止。禁止在未确认时执行任一种模式，也禁止 Agent 自动降级为 full。

**深度由 Agent 代选或未确认即执行**：可向用户转述常见取舍（大库初扫往往先选 1，全量精读选 3），但不得替用户勾选；用户选定后再执行。

**未确认即采用 `--since` 或默认输出路径**：候选时间戳与默认路径须展示并请用户确认（或用户显式给出字面量）后再写入命令；禁止静默采用。

**增量基线时间戳来源错误**：增量模式的基线时间戳唯一来源是 `indexing-log.jsonl` 最后一条记录的 `indexing_finished_at`，不是 `changes-index.json` 的 `baseline_time`。

---

## 扫描与采集

**索引构建产物目录**：严格按排除列表过滤 `target/`、`build/`、`node_modules/` 等，输出目录自身也须排除，否则索引膨胀充斥无意义条目。

**把未读文件写成已索引内容**：零幻觉原则——只索引实际读取的内容；未读路径标注 `[未索引]` 并说明原因，归入 §八。

**深度 3 抽样跳读**：用户已确认 `depth=3` 时，不能只读每模块少量示例文件或跳过「相似」的同级文件。须在排除规则内系统遍历并尽量读全；无法读完的路径必须在 §八 明示。详见 [reference/scan-spec.md](reference/scan-spec.md)「深度 3 应读尽读准则」。

**增量模式清空未变更章节**：增量更新只合并变更文件对应的条目，未变更部分保持原样，不重写整个 INDEX_GUIDE.md。

**同一文件归入多个章节**：每个文件只归入一个最匹配的章节，跨章节引用以超链接形式体现，不重复内容（MECE 原则）。

---

## 九章结构生成

**留有空章节**：确实无内容的小节标注 `[未索引]` 并说明原因（如「当前项目无消息队列组件」），归入 §八排除列表，不留空白。

**条目信息密度不达标**：每条目 15-30 字，包含功能描述、关键属性或路径，禁止「用户服务」「订单模块」等空洞标签。

**路径使用绝对路径或错误相对路径**：一律使用项目根相对路径（如 `src/main/java/com/example/Service.java`），不用 `/Users/xxx/...` 或 `../src/...`。

**版本号、配置值凭印象填写**：§七配置中心的版本号或配置值必须来自实际读取的 `pom.xml`、`application.yml` 等文件，不猜测。

---

## 输出与日志

**输出路径未确认或优先级混淆**：若需使用约定优先级（用户指定 > `./application/` > `./doc/` > `./INDEX_GUIDE.md`），须将解析结果展示给用户并获确认后再写入；用户已显式给出路径则以用户路径为准。

**未追加操作日志**：每次执行完毕必须追加（append）一条完整日志记录到 `changelogs/indexing-log.jsonl`，包含 `indexing_finished_at`；不得覆盖历史记录。

**日志字段缺失**：严格按日志格式输出所有必需字段：`timestamp`、`data_mode`、`read_mode`、`output_path`、`indexed_files`、`duration_ms`、`indexing_finished_at`。

---

## 与上下游协作

**增量模式跳过 docs-change**：增量模式必须先通过 `docs-change` 获取 `changes-index.json`，以变更文件列表驱动扫描范围，不能直接扫描全部文件。

**与 agent-guide 产出的目录树矛盾**：agent-guide 生成 README 时以本 INDEX §二为唯一来源；本 Skill 更新 §二后需通知 agent-guide 同步，保持两处一致。

---

## 快速自查清单

- [ ] 扫描模式与深度已由用户确认，非 Agent 默认定参
- [ ] `--since`、默认 `--output` 等已展示并确认（或用户显式给出）
- [ ] 九章均有实质内容，无空章节（空内容标注 `[未索引]`）
- [ ] 每条目 15-30 字，无占位符和空话
- [ ] 所有路径为项目根相对路径，可访问
- [ ] 版本号、配置值来自实际读取的文件
- [ ] 每个文件只归入一个章节（MECE）
- [ ] 操作日志已追加到 `changelogs/indexing-log.jsonl`，字段完整
- [ ] 增量模式未清空未变更章节
- [ ] 构建产物目录已排除
- [ ] 未读内容未写成已核实事实（零幻觉）
- [ ] 若深度为 3：已按应读尽读准则遍历，无抽样跳读
