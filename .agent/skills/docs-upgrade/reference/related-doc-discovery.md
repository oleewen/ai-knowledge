# 关联发现（引用链检索）

在未声明「仅改本文件」时，对本轮已修改的每个文件分别做入站、出站检索。默认深度：**一层入站 + 一层出站**；用户要求「全链」时再扩展并提示风险。

与「无直接链接但需对齐同一术语」的补充手段见 [semantic-keyword-discovery.md](semantic-keyword-discovery.md)。

---

## 出站（该文件引用了谁）

### Markdown / 纯文本

读全文，收集：`[text](path)`、`[text](path#anchor)`、反引号内 `*.md` 或 `docs/...` 等裸路径，将相对路径解析为仓库内路径（相对当前文件目录）。

### 源代码中的注释

收集注释内显式路径、文件名、`@path`、`TODO(file:...)`、以及指向文档的 URL/相对路径字符串。`import`/`require`/`include` 等若指向仓库内文件，仅当本次变更影响「如何引用该模块」的叙述时再去改目标文件注释，避免无关扰动。

---

## 入站（谁引用了该文件）

对主文件的每一种可检索键检索（至少包含）：

| 键类型 | 示例 | 工具 |
|--------|------|------|
| 仓库内相对路径 | `docs/solutions/X.md` | `rg -F` |
| 从某目录起算路径 | `solutions/X.md` | `rg -F` |
| 文件名 | `X.md` | `rg -F`（同名多文件时列出候选请用户选） |
| 代码中字符串 | `.java` 内 `"../docs/X.md"` | `rg -F` |

排除 `target/`、构建产物、二进制目录（与仓库 AGENTS 约定一致）。命中过多时优先 `docs/`、`application/` 等文档前缀，再请用户缩小范围。

---

## 同类表述

指与主变更**同一术语定义、同一列表项、同一摘要句、或明显复制的段落**。不要强行统一仅碰巧用词相近但含义不同的段落。

---

## 链式深度

默认一层入站 + 一层出站；扩展 BFS 前提示工作量与误改风险。

完成入站/出站后，若仍有散落同义表述，再按 [semantic-keyword-discovery.md](semantic-keyword-discovery.md) 做关键词检索。
