# 关键词与语义对齐（无直接链接时）

当文件之间没有 Markdown 链接或注释内显式路径，但用户要求「全库对齐同一说法」或主变更涉及术语、列表项、摘要句等可能多处重复时，在引用链之外增加本检索路径。

---

## 检索词范围（须覆盖）

除字面一致的片段外，主动纳入下列词面分别检索（可分批 `rg`，避免一次模式过于宽泛）：

| 类型 | 说明 | 示例 |
|------|------|------|
| **同义词** | 可互换且在本仓库语境下指向同一概念 | 「登录」与「登入」若用户要求统一 |
| **近义词** | 含义接近、易与主变更混写 | 「实现」与「达成」等，仅在用户意图为统一表述时使用 |
| **中英文同义** | 同一概念的中英混写、缩写与全称 | `API` 与「接口」、中文标题与英文 slug 并存处 |

同一轮主变更应维护一张小型候选词表（来自用户原话、主文件标题/术语表、常见配对），对表中每一项做检索后再做语义判断是否改写。

---

## 何时启用

- 用户明确要统一术语、品牌名、API 名、配置键名等
- 主文件中出现的短语在其它文档或注释里以同义、近义、略写或另一种语言形式重复出现
- 链式一层检索未覆盖，但 `rg` 显示仍有高置信命中

---

## 检索策略

1. **高置信短语**：从主变更抽取 1～3 个不易误伤的连续片段（专有名词、带括号的正式名称、错误拼写修正对等），使用 `rg -F` 或项目约定的搜索。
2. **扩展同义/近义/中英对应**：为旧概念列出候选词面，逐项检索；命中过多时按目录优先级缩小（先 `docs/`、`{DOC_DIR}/`，再其它），或请用户加排除路径。
3. **语义判断**：仅当命中处与主变更明显是同一概念、同一列表项或复制段时才改。碰巧用词相近但含义不同的段落不改；近义词边界不清时走 SKILL 正文的不确定项门禁，请用户选是否替换。

---

## 与引用链的配合顺序

**先完成主文件修改 → 再跑引用链（入站/出站一层）→ 再对仍存疑或用户点名的术语跑关键词检索。**

避免在未读主文件前全库替换导致上下文错误。

---

## 风险控制

- 短字符串（少于 3～4 个有意义字符）或常见英文词：易误伤，须人工核对或缩小文件范围
- 生成代码、锁文件、二进制目录：排除构建产物目录（与仓库 AGENTS/忽略约定一致）
