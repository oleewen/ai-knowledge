# 设计说明：将 `.ai` 目录重命名为 `.agent`（评估与实施要点）

**日期**：2026-04-07  
**状态**：设计记录（未实施变更）  
**范围**：`ai-knowledge` 仓库内目录命名策略与可选迁移方案

---

## 1. 背景与现状

`.ai/` 在本仓库中作为 **AI 协作控制层**：承载 `rules/`、`skills/`（Slash 技能）及共享 Bash 库 `scripts/`（如 `sdx-doc-root.sh`、`sdx-validate-bootstrap.sh`）。该路径在以下位置被**硬编码或强约定**：

- **仓库根探测**：`/.ai/scripts/sdx-doc-root.sh` 存在性用于向上解析仓库根。
- **初始化链**：`scripts/docs-init.sh` 从 `${repo_root}/.ai/skills`、`/.ai/rules` 拷贝到目标 Agent 目录，并在文件内容中将 **`.ai/`** 替换为目标 Agent 前缀（如 `.cursor/`）。
- **文档与元数据**：根目录与 `system/`、`applications/` 下大量 Markdown 相对链接与 `*_meta.yaml` 中的路径引用。
- **索引产物**：根目录 `INDEX_GUIDE.md`、`system/INDEX_GUIDE.md` 等含批量 `.ai/` 路径枚举（生成型文档需按项目流程更新）。

粗略量级：**数百处**字面量引用；全量替换需脚本扫描与人工抽查，不能仅靠单次 `sed`。

---

## 2. 目标与非目标

**目标**

- 记录将 `.ai` 改为 `.agent` 的**收益、成本、风险**与**可选实施方案**。
- 若决议实施，为后续 `writing-plans` 阶段提供**单一事实来源**的检查清单。

**非目标**

- 本文**不**包含对已落地目录的实际重命名或批量替换（实施须单独 PR）。
- 本文**不**变更 `AGENTS.md`、根 `README.md` 等契约文件，除非变更已获批准并纳入实施任务。

---

## 3. 方案对比

| 方案 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| **A. 保持 `.ai/`** | 不更名 | 零迁移成本；与现有 bootstrap 用户、站内链接一致 | 名称偏泛，「AI」与「规则+技能目录」的对应依赖文档解释 |
| **B. 重命名为 `.agent/`** | 目录改为 `.agent/`，全量更新引用与脚本 | 语义更贴近「Agent 规则与 Slash 技能」；与 `.cursor/`、`.trea/` 等并列时叙事更清晰 | 一次性工作量大；对外部已克隆仓库、书签、CI 中写死路径可能**破坏性**；需更新 `docs-init` 源路径与 Perl 替换模式 |
| **C. 过渡期双路径** | 例如符号链接 `.ai` → `.agent` | 短期兼容旧链接 | Git 下 symlink 跨平台与 CI 风险；双路径增加心智负担，不宜长期保留 |

**推荐（默认）**

- 若仅为「名称更好听」或轻微偏好：**优先方案 A**，成本收益比更优。
- 若产品/文档层面已明确要求统一使用「agent 层」命名，且接受破坏性变更与迁移说明：**采用方案 B**，并发布**显式变更说明**（changelog / README 提示 / 可选版本标签策略由维护者定）。

不推荐方案 C 作为长期状态。

---

## 4. 若实施方案 B：实施要点

### 4.1 目录与职责

目标结构与原 `.ai/` **同构**：

```text
.agent/
├── README.md
├── rules/
├── skills/
└── scripts/
```

职责不变：仍为本仓库内「规范 + Slash 技能 + 技能共享脚本」的权威位置。

### 4.2 必须修改的技术点

1. **仓库根探测**（`sdx-doc-root.sh`、`sdx-validate-bootstrap.sh`）  
   - 将「存在 `/.ai/scripts/sdx-doc-root.sh`」改为「存在 `/.agent/scripts/sdx-doc-root.sh`」（或抽象为单一常量/变量，避免再次硬编码目录名）。

2. **`scripts/docs-init.sh`**  
   - 源目录：`/.ai/skills`、`/.ai/rules` → `/.agent/skills`、`/.agent/rules`。  
   - 内容替换：将 `s{\.ai/}` 类替换调整为 `s{\.agent/}`（或等价变量），使拷贝到 `.cursor/` 等目标时语义仍为「中央库 agent 层 → 用户 Agent 目录」。

3. **技能内脚本与模板路径**  
   - 各 `validate-*.sh` 中 `TEMPLATE=".ai/skills/..."` 等改为 `.agent/...`。  
   - `SKILL.md` 与 gotchas 中的示例命令同步更新。

4. **文档与 YAML**  
   - `README.md`、`AGENTS.md`、`system/**/*.md`、`applications/**/*.md`、`*_meta.yaml` 中指向 `.ai/` 的链接与路径列表。

5. **索引**  
   - 根 `INDEX_GUIDE.md` 与 `system/INDEX_GUIDE.md`：按仓库约定执行 `/docs-indexing` 或等效流程**重生成**，避免手工漏改。

### 4.3 验证清单

- 在仓库根执行各 `validate-*.sh` 及 `agent-guide` 的 `validate-guide.sh`（路径以 `.agent/` 为准）。
- 对 `docs-init.sh` 做一次**干跑或临时目标目录**试装，确认拷贝与替换结果中**不再出现**错误的 `.ai/` 残留（除非刻意保留的说明性文字）。
- 全文搜索：对 `.ai/` 做剩余命中审查（排除无关内容，如 Java 包名 `com.ai.*`、正文「Claude.ai」等）。

### 4.4 对外与迁移说明

- 在 `README.md` 或 `system/changelogs/` 中增加**简短破坏性说明**：旧路径 `.ai/` 已废弃，请更新书签与自动化脚本。
- 若有对外 raw 脚本 URL（如 GitHub raw），列出需更新的链接。
